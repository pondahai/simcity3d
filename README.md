## 專案起源

本專案源自最初的一句提示詞:

> 網頁版模擬城市第一代 的功能都復刻 但是改成立體成像 提供兩個視角 第一是鳥瞰 第二是走在街上 無論視角 都要保持模擬城市 以上初步條件你是否理解

也就是三個核心條件:

1. **復刻初代《模擬城市》(1989)的完整功能** — RCI 分區生長、電網、道路可達性、稅務財政、災難系統。
2. **立體成像** — 以 Three.js 呈現低多邊形 3D 城市。
3. **雙視角,且模擬不中斷** — 鳥瞰(施政視角)與街景(第一人稱走在街上,按 V 切換);無論在哪個視角,城市模擬持續運轉。

## 快速開始

```
python -X utf8 build.py    # 將 part1~part5 組裝成 立體模擬城市.html(Windows 必須加 -X utf8)
```

用瀏覽器開啟輸出的 `立體模擬城市.html` 即可遊玩(需網路載入 three.js CDN)。

## 操作

- **鳥瞰**:拖曳平移(中鍵在任何工具下都是平移)、右鍵/Ctrl 旋轉、滾輪縮放;左側工具列施工。
- **街景**:按 `V` 進入/離開;WASD/方向鍵移動、Shift 跑步、拖曳環顧;手機有虛擬搖桿。街景為觀光模式,不可施工。
- **存檔**:每年度結算與關閉頁面時自動存到瀏覽器(localStorage),下次開同一網址自動接續;「🆕」彈窗內可匯出/匯入 JSON 存檔跨裝置備份。

## 模擬運作原理(目前程式的實際做法)

### 世界與時間

城市是 64×64 的格網,每格一個 cell:`{t 地格類型, lvl 發展等級, pow 供電, ax/ay 大建築錨點, fire 燃燒計時, br 橋梁旗標, wr 電線交叉旗標, v 隨機種子}`。時間以「月」為模擬節拍:主迴圈依速度檔(暫停/2400/1100/380 毫秒一個月)呼叫 `simMonth()`,一次節拍完成供電→生長→需求→火災→財政→災難→統計的完整結算。

### 電力系統

`computePower()` 從每座電廠(火力容量 220、核能 550)做 BFS 洪水填充,導電體為:電線、道路/鐵路上的交叉電線(`wr`)、分區格、大型建築。同一連通元件內的電廠容量合併;分區與建築每格消耗 1 單位容量,電線不耗電,超出容量的格子斷電(blackout)。電力狀態在施工、拆除、新開局、災難後都會**即時重算**,不必等月節拍。

### 道路可達與分區生長

`computeAccess()` 從所有道路/鐵路格 BFS 擴散 3 格,標記「有路可達」。每月每個分區格擲骰決定升降級(lvl 0~5):

**道路 vs 鐵路**:兩者對「可達性」完全等價;差異在於——鐵路造價 $20(道路 $10)、維護 0.5/格(道路 0.18),但鐵路格的交通壅塞值只有道路的 30%,且**不產生交通污染**(視為電氣化),適合高密度區降低污染;鐵路上另有火車動畫行駛。

- **升級條件**:有電 + 3 格內有路 + 該類需求 > 0。機率 = 0.10 + 需求×0.45 + 地價加成 − 環境懲罰(污染或犯罪 > 55 各扣 0.35)。
- **衰退條件**:斷電(月衰退機率 0.35)、需求 < −0.25 或環境太差(0.12)。

人口 = 住宅總等級 × 22。

### RCI 需求模型

- **住宅**:由就業(商業+工業等級和)拉動,住宅供給過多則下降;體育場 +0.12。
- **商業**:由住宅人口拉動。
- **工業**:由「對外市場」`ext` 拉動——`ext` 每月緩慢成長,海港 +50%、機場 +70% 加速。
- 三者都受稅率懲罰:稅率每高於 7% 一個百分點,需求扣約 0.055(工業 ×1.2)。

### 統計圖層(每月重算,Float32Array + 盒狀模糊)

- **交通**:已發展分區向周圍 4 格輻射(等級×7),道路格保留全額、鐵路格降至 30%、其他格衰減至 25%。
- **污染**:工業(18+等級×11)、火力廠(65)、機場(28)、火場(45)、瓦礫(8)、交通×0.22(鐵路格免計),模糊 3 次。
- **警/消覆蓋**:從警局/消防局向外放射狀輻射,半徑隨撥款比例縮放(基準 13 格)。
- **地價**:基準 28,近水 +9、公園 +14、體育場 +12、市中心微加成,再扣污染×0.45。
- **犯罪**:人口密度(等級×14)− 地價×0.35 − 警力×0.6。

### 財政(逐月)

收入 = (住宅×1.0 + 商業×1.6 + 工業×1.6) × 稅率 × 0.32;支出 = (道路×0.18 + 鐵路×0.5)×交通撥款 + 警局/消防局各 24×撥款比例。撥款比例可在預算彈窗調整:警/消撥款影響覆蓋半徑;交通撥款低於 100% 時,每格道路/鐵路每月有 (1−撥款)×1.2% 機率年久失修損毀成瓦礫(橋梁還原為水域)。

### 火災與災難

火場每月向四鄰蔓延(基礎機率 0.32,消防覆蓋可壓到 0.03),燒 2 個月以上開始塌成瓦礫。六種災難:火災、洪水(沿岸格變瓦礫)、龍捲風與巨獸(動態實體,每 0.22 秒移動一步、輾壓沿途)、地震(全圖隨機 90 格破壞+起火)、空難(3×3 起火)。人口 > 800 後每月有 0.6% 機率自動觸發(可關閉)。

### 特殊地格規則

- **跨河橋**:道路/鐵路/電線可蓋在水上,造價 5 倍,設 `br` 旗標;任何破壞路徑都會還原成水域。
- **電線交叉**:電線與道路/鐵路可互相跨越,以 `wr` 旗標疊加(地格類型不變),推土機先拆電線再拆路;破壞路徑會一併清除。
- **平交道**:道路與鐵路可互相跨越,統一存為道路格 + `rl` 旗標;車流與火車都會通過,維護費同時計道路與鐵路,推土機先拆鐵路再拆路。橋面與已有電線交叉的格子不可再疊平交道(反之亦然);破壞路徑會一併清除。

### 渲染

Three.js InstancedMesh(自訂 shader 做逐實例顏色),模擬狀態變動時設 `dirty`,下一幀整批重建地面與建物;車流沿道路、火車(車頭+車廂)沿鐵路獨立步進。鳥瞰與街景共用同一份模擬狀態,切視角不影響模擬運轉。
